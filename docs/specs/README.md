# Specs

Design documents for features and restructures. Specs describe *what* is
being built and *why* — scope, requirements, architecture, data models, and
the reasoning behind design decisions. Active specs live here while the
design is being worked on or the implementation is underway; completed specs
move to `.archive/specs/` once the work is done and any corresponding ADR is
committed.

Specs answer "what are we building?". Workplans (`docs/workplans/`) answer
"how are we building it?". ADRs (`docs/adr/`) are the permanent record of
decisions made along the way.
